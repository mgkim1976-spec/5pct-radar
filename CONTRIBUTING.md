# 기여 가이드

`5pct-radar` 에 기여를 환영합니다. 다음 영역이 특히 도움이 됩니다.

## 환영하는 기여

### 1. 그룹 구조 매핑 정확도 개선

- 다단 그룹 (2-hop 이상) 매핑 알고리즘
- 자회사 명칭 정규화 규칙 보강
- 외국법인 (해외 SPV) 매핑

### 2. 시나리오 분류 정교화

- 새 카테고리 추가 (예: 적대적 인수, 우호적 합병)
- 분류 확률 검증 (실제 catalyst 발생률 대비)
- LLM prompt 개선

### 3. 데이터 소스 확장

- 분기보고서 임원·이사회 정보 연결
- 인수 후 후속 공시 자동 추적 (catalyst chain)
- 외국계 펀드 데이터 통합

### 4. 자동 테스트 보강

- `tests/` 에 새 테스트 케이스 추가
- 도메인 규칙의 가장자리 사례
- 그룹 구조 매핑 unit test

## 기여 절차

1. 저장소를 본인 계정으로 복사 (Fork)
2. 새 브랜치 생성: `git checkout -b feature/원하는-기능`
3. 코드 변경 + 테스트 작성
4. 코드 검사 통과 확인: `ruff check src tests`
5. self-test 통과 확인: `python -m five_pct_radar --self-test`
6. Commit + Push
7. Pull Request 생성

## 코드 스타일

- Python 3.11 이상
- `ruff check` 통과
- 모든 외부 공개 함수에 짧은 한국어 또는 영어 설명
- 분류 룰 변경 시 [STORY.md](STORY.md) 또는 [CHANGELOG.md](CHANGELOG.md) 에
  *왜* 변경했는지 기록

## 절대 받지 않는 변경

다음 변경 제안은 그 어떤 형태로도 받지 않습니다:

1. **모든 보고서 §7 "사람 검증 필수 항목" 섹션 제거** — 자동화의 정직성 핵심
2. **미래 수익률을 *예측* 한다는 주장** — EV 분포는 *시나리오 가중 추정* 일 뿐.
   *예측* 표현 금지
3. **투자 권유성 텍스트** ("이 종목 사세요" 류)
4. **`.env` 파일 또는 비밀 정보 commit**
5. **LLM evidence 필드 제거** — 본문 인용 강제는 환각 검증 핵심
